Что делает конвертер

ino2ubi преобразует Arduino-скетчи (.ino) в пользовательские блоки (.ubi) для программы FLProg. Блок можно импортировать в FLProg и использовать как обычный пользовательский блок.

Пошаговая инструкция

  1. Загрузите .ino (кнопка «Загрузить .ino») или вставьте код Arduino в поле слева.
  2. Нажмите «Парсить код» — конвертер извлечёт переменные, функции, #include и #define.
  3. Во вкладках «Переменные» и «Функции» при необходимости отредактируйте настройки (двойной клик по строке).
  4. Задайте название и описание блока.
  5. Нажмите «Сгенерировать .ubi и скачать» — файл сохранится на компьютер.
  6. Импортируйте .ubi в FLProg (меню библиотеки блоков).

Роли переменных

  • variable — внутренняя переменная, не отображается на блоке.
  • input — вход (слева на блоке), получает данные от других блоков.
  • output — выход (справа на блоке), передаёт данные другим блокам.
  • parameter — настраиваемый параметр, задаётся в свойствах блока.

Определение ролей в коде

Роли можно задать комментариями после объявления переменной:

int sensorValue = 0;  // in — вход
int result = 0;       // out — выход
int threshold = 100;  // par — параметр

Для #define роль параметра задаётся комментарием //par.

Примечания

  • Комментарий в начале скетча (// или /* */) используется как описание блока.
  • Функции setup() и loop() автоматически помещаются в соответствующие секции блока.
  • Дополнительные функции отображаются во вкладке «Функции»; тело можно редактировать двойным кликом.
  • Опция «Вход En» оборачивает код в loop в условие if(En) { … } — блок будет выполняться только при активном входе En.
